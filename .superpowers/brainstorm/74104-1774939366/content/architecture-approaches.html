RAG Document Layer — Architecture Approaches

How should the document pipeline integrate with FORGE's existing flow?

A

Upload-Time Processing (Recommended)

Process documents immediately on upload. Extract text, chunk, embed, and store in Upstash Vector — all in the upload API route. At analysis time, embed the scenario query and retrieve top-k relevant chunks to inject alongside Perplexity research.

Upload Flow:
Browser → POST /api/documents → extract text (pdf-parse) → chunk (~500 tokens each) → embed (OpenAI) → store in Upstash Vector + metadata in Vercel Blob
Analysis Flow:
Scenario text → embed query (OpenAI) → search Upstash Vector (top 10 chunks) → inject into Claude prompt alongside Perplexity research → bilingual analysis

Pros

  • Analysis runs fast — chunks already embedded
  • Upload gives immediate feedback (doc processed)
  • Clear separation of concerns

Cons

  • Upload takes a few seconds (embed all chunks)
  • Two API routes to build (upload + analysis modification)
B

Analysis-Time Processing

Store raw files in Vercel Blob on upload. When analysis runs, extract + chunk + embed + search all in the analysis API route. Simpler upload, but heavier analysis step.

Upload Flow:
Browser → POST /api/documents → store raw file in Vercel Blob → done
Analysis Flow:
Scenario text → fetch all docs from Blob → extract text → chunk → embed all chunks + query → cosine similarity → inject top chunks → Claude analysis

Pros

  • Upload is instant (just file storage)
  • Single processing pipeline

Cons

  • Every analysis re-processes all documents (slow, wasteful)
  • Embedding cost multiplied by every analysis run
  • Analysis latency increases significantly
